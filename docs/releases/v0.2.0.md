# DevMemory v0.2.0 release notes draft

> Status: draft
> Target: v0.2.0
> Scope: local-first developer memory lifecycle, local AI/RAG usability, diagnostics, and demo readiness.

## Overview

DevMemory v0.2.0 is a major portfolio-ready iteration focused on making the tool more complete, easier to use, and easier to demonstrate.

This release improves the core memory lifecycle, strengthens local AI/RAG workflows, adds discovery commands, introduces general diagnostics, and provides an isolated demo experience for GitHub visitors, reviewers, and future contributors.

The main goal of this release is to move DevMemory from a working local-first CLI prototype to a more complete developer tool that can be demonstrated publicly.

## Highlights

### Complete memory lifecycle

DevMemory now supports a more complete local memory lifecycle:

```bash
devmemory add
devmemory list
devmemory show <memory-id>
devmemory search <query>
devmemory edit <memory-id> [options]
devmemory delete <memory-id> [--yes]
```

This makes the tool more practical for daily usage because saved memories can now be corrected, extended, and removed without manually editing the JSON storage file.

### Persistent AI configuration

DevMemory now supports persistent AI/RAG configuration through:

```bash
devmemory config show
devmemory config set <key> <value>
devmemory config reset
```

Configuration is stored locally and follows this precedence:

```text
Environment variables > ~/.devmemory/config.json > default values
```

This makes local AI usage easier because users no longer need to pass environment variables before every command.

### Memory edit command

A new command was added:

```bash
devmemory edit <memory-id> [options]
```

It supports updating scalar fields and collection fields, for example:

```bash
devmemory edit <memory-id> --title "Updated title"
devmemory edit <memory-id> --solution "Updated implementation notes"
devmemory edit <memory-id> --add-tag rag
devmemory edit <memory-id> --remove-tag test
devmemory edit <memory-id> --add-file src/Example.cs
devmemory edit <memory-id> --add-test ExampleTests
```

When a memory is edited:

```text
the primary JSON memory is updated
the previous Markdown export is removed
a new Markdown export is generated
```

If the memory was already indexed into the vector store, the user is informed that the derived vector index should be rebuilt.

### Memory delete lifecycle

The delete command has been extended into a more complete lifecycle operation:

```bash
devmemory delete <memory-id>
devmemory delete <memory-id> --yes
```

The command now removes:

```text
the memory from primary JSON storage
the derived Markdown export
the Qdrant vector point when a vector store is configured
```

Qdrant cleanup is best-effort. If vector cleanup fails, the primary delete still succeeds and DevMemory prints a clear warning.

This keeps JSON as the source of truth while treating Markdown and Qdrant as derived artifacts.

### Related memories

A new semantic discovery command was added:

```bash
devmemory related <memory-id>
devmemory related <memory-id> --limit 3
devmemory related <memory-id> --show-preview
```

This command finds memories that are semantically related to another memory.

It reuses the existing embedding and vector search pipeline and excludes the source memory from the result list.

This is one of the most important AI-assisted discovery features in DevMemory because it helps reconnect related work across different tasks, branches, and technical areas.

### Timeline

A new timeline command was added:

```bash
devmemory timeline
devmemory timeline --project DevMemory
devmemory timeline --area AI
devmemory timeline --tag rag
devmemory timeline --limit 10
```

The timeline prints saved memories grouped chronologically.

This is useful to understand project evolution over time and is especially helpful for demos, screenshots, and retrospectives.

### General doctor command

A new general diagnostics command was added:

```bash
devmemory doctor
```

This command checks the overall local DevMemory environment:

```text
storage readability
Markdown directory
persistent configuration file
AI/RAG runtime configuration
Git availability
memory count
```

It complements `devmemory ai-doctor`, which remains focused on local AI runtime diagnostics such as Ollama and Qdrant.

### Isolated local demo script

A new isolated demo script was added:

```bash
./scripts/demo-local.sh
```

The demo creates a temporary `DEVMEMORY_HOME`, seeds sample memories, runs core commands, and deletes the temporary data automatically.

It demonstrates:

```text
doctor
list
search
show
timeline
edit
graph-export
graph-view
optional AI indexing
optional semantic search
optional related memories
optional RAG
```

The demo does not touch the real `~/.devmemory` directory.

Users can keep demo data for inspection with:

```bash
DEVMEMORY_KEEP_DEMO_HOME=true ./scripts/demo-local.sh
```

### Demo documentation

A dedicated demo guide was added:

```text
docs/demo.md
```

The guide explains:

```text
why the demo is isolated
how to run it
how to keep demo data
what each demo step shows
how the optional AI section works
how to troubleshoot Ollama and Qdrant issues
```

This improves onboarding for people opening the repository from GitHub or LinkedIn.

## Technical improvements

### Local-first architecture reinforced

The release reinforces the local-first architecture:

```text
JSON remains the primary source of truth
Markdown remains a portable derived artifact
Qdrant remains a derived vector index
Ollama/Qdrant are optional for AI features
Core memory workflows work without AI services running
```

This keeps the tool useful even when local AI dependencies are not available.

### More robust CLI behavior

The CLI now provides clearer behavior for:

```text
missing memory ids
invalid GUIDs
unknown options
invalid limits
not configured AI runtime
missing vector store adapter
failed vector cleanup
empty result sets
```

### More complete test coverage

The test suite was expanded to cover:

```text
persistent AI configuration
memory edit behavior
memory delete behavior
Markdown cleanup
Qdrant vector deletion
related memories
timeline output and filters
general doctor diagnostics
isolated demo script validation
```

At the time of this draft, the suite contains more than 200 tests.
